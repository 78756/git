## 硬件接线指南

### A4988 驱动板引脚定义

```
    ┌─────────────┐
    │   A4988     │
 IN1├─────────────┤OUT1
 IN2├─────────────┤OUT2
 IN3├─────────────┤OUT3
 IN4├─────────────┤OUT4
    │             │
 GND├─────────────┤GND
+5V ├─────────────┤+5V (VMOT)
 EN ├─────────────┤(使能)
DIR ├─────────────┤(方向)
STP ├─────────────┤(步进/脉冲)
    └─────────────┘
```

### 28BYJ-48 步进电机内部接线

```
28BYJ-48 是四相步进电机，通常有5条线（1条公共线 + 4条控制线）

接线颜色（可能因厂商而异）：
  - 红色：公共端 (+5V)
  - 黄色：A 相
  - 粉色：B 相
  - 橙色：C 相
  - 棕色：D 相

连接到 A4988：
  红色 → A4988 的 +5V (VMOT)
  黄色 → OUT1
  粉色 → OUT2
  橙色 → OUT3
  棕色 → OUT4
```

### 完整接线示意图

```
┌─────────────┐
│   Arduino   │
│             │
│  GND────────┼────────────┐
│  2 (X_DIR)  │            │
│  3 (X_STEP) │            │
│  4 (X_EN)   │            │
│             │            │
│  5 (Y_DIR)  │            │
│  6 (Y_STEP) │            │
│  7 (Y_EN)   │            │
│             │            │
│  8 (Z_DIR)  │            │
│  9 (Z_STEP) │            │
│ 10 (Z_EN)   │            │
│  GND────────┼────┐       │
│  5V 或 GND  │    │       │
└─────────────┘    │       │
                   │       │
      ┌────────────┴────┐  │
      │                 │  │
   ┌──────┐         ┌───┴──┴────┐
   │A4988 │ ← DIR ──┤ Arduino   │
   │  X轴 │ ← STP ──┤ (3根线)   │

注意：
1. 必须使用 GND 连接
2. 方向、步进、使能引脚通过杜邦线连接
3. 每个 A4988 需要单独的 OUT1-4 连接到电机
```

### 电源连接建议

**方案1：USB供电（低功率，推荐用于开发）**
```
Arduino USB 5V → A4988 的 VMOT
Arduino GND    → A4988 的 GND
```

**方案2：外部电源（推荐用于实际应用）**
```
外部 5V 电源 + → A4988 的 VMOT
Arduino 5V    → A4988 的 VMOT（可选，共享电源）
外部 5V 电源 - → A4988 的 GND
Arduino GND   → A4988 的 GND（必须共地）
```

## A4988 跳线设置

### 微步配置（MS1, MS2, MS3 跳线）

在 A4988 上找到 MS1, MS2, MS3 三个跳线位置

```
MS1  MS2  MS3  │  步进模式    │  步距
────────────────┼──────────────┼─────────
 无   无   无   │  全步（1/1）  │ 5.625°
 有   无   无   │  半步（1/2）  │ 2.8125°
 无   有   无   │  1/4 微步    │ 1.4°
 有   有   无   │  1/8 微步    │ 0.7°
 无   无   有   │  1/16 微步   │ 0.35° ← 推荐
 有   无   有   │  1/32 微步   │ 0.176°
```

**推荐配置：**
- MS1 放上（有跳线）
- MS2 不放（无跳线）
- MS3 放上（有跳线）
- 结果：1/16 微步模式

## 故障排除

### 问题1：电机完全不动

**诊断步骤：**
```cpp
// 在 Arduino IDE 的串口监视器中运行这个代码段
void test_pins() {
  // 测试 X轴 DIR 引脚
  digitalWrite(2, HIGH);
  delay(1000);
  digitalWrite(2, LOW);
  delay(1000);
  
  // 测试 X轴 STEP 引脚
  digitalWrite(3, HIGH);
  delay(500);
  digitalWrite(3, LOW);
  delay(500);
  
  // 测试 X轴 ENABLE 引脚
  digitalWrite(4, LOW);  // 启用
  delay(1000);
  digitalWrite(4, HIGH); // 禁用
}
```

**检查清单：**
- [ ] 5V 电源是否供电？
- [ ] 杜邦线连接是否牢固？
- [ ] A4988 上的 ENABLE 跳线是否正确？
- [ ] 电机线是否连接正确？

### 问题2：电机转向错误

**解决方案：**
修改代码中的方向设置
```cpp
// 方法1：改变方向逻辑
digitalWrite(dir_pin, LOW);   // 改为 LOW

// 方法2：交换电机接线
// 将电机的 A 和 C 相互交换
// 或将 B 和 D 相互交换
```

### 问题3：电机抖动/噪音

**原因和解决方案：**

| 症状 | 原因 | 解决方案 |
|------|------|---------|
| 高频噪音 | 脉冲太快 | 增加 `pulse_delay` 值 |
| 明显抖动 | 功率不足 | 检查电源，可能需要更强的电源 |
| 间歇性噪音 | 接线松动 | 重新检查所有连接 |

### 问题4：某个轴不工作，其他轴正常

**诊断：**
1. 检查该轴的引脚是否正确
2. 检查 A4988 驱动板是否正常
3. 尝试交换电机测试（看是否是电机问题）

### 问题5：无法通过串口通信

**检查清单：**
```cpp
// 在 setup() 中验证
Serial.begin(9600);
Serial.println("通信测试");

// 你应该在串口监视器中看到 "通信测试"
// 如果没有看到：
// - 检查波特率是否设置为 9600
// - 检查 COM 端口是否选择正确
// - 检查 USB 线是否牢固
```

## 电源计算

```
总功耗 = 电机功耗 × 数量 + A4988 功耗 × 数量

单个 28BYJ-48：约 200mA @ 5V = 1W
单个 A4988：约 100mA @ 5V = 0.5W

三轴系统：
  电机：200mA × 3 = 600mA
  驱动板：100mA × 3 = 300mA
  总计：约 1A（最小推荐 1.5A 电源）
```

推荐使用 **2A 以上** 的 5V 电源以保证稳定性。
